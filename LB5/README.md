# Hotel Room Booking System — Hexagonal Architecture

## 1. Назва роботи
Проєктування програмних систем: Система бронювання номера в готелі (Варіант 6).

## 2. Опис use case
Система дозволяє користувачеві забронювати номер у готелі на визначені дати.
Основний сценарій перевіряє:
- Чи існує такий номер у базі.
- Чи доступний номер для експлуатації загалом.
- Чи немає перетину дат із вже існуючими бронюваннями для цього номера.
  Після успішних перевірок система генерує бронювання та зберігає його.

## 3. Обрана архітектура
У проєкті використано **Hexagonal Architecture** (відома також як Ports and Adapters).

## 4. Обґрунтування вибору
Цей підхід було обрано, оскільки він дозволяє повністю ізолювати бізнес-логіку бронювання від зовнішніх технологій (баз даних, користувацького інтерфейсу). Завдяки цьому ми можемо легко покрити бізнес-правила Unit-тестами, а в майбутньому змінити in-memory сховище на реальну БД (наприклад, PostgreSQL) чи додати REST API без жодних змін у ядрі застосунку.

## 5. Структура проєкту
- `domain/`: Сутності та правила, що не залежать від інших шарів (`Room`, `Booking`, `DateRange`).
- `application/`: Сценарії використання (`BookingService`) та інтерфейси-порти (`BookRoomUseCase`, `RoomRepository`).
- `infrastructure/`: Технічні реалізації взаємодії з системою (адаптери).
- `Main.java`: Композитний корінь (Composition Root), де збираються всі залежності.

## 6. Розподіл відповідальностей
- **Domain Logic:** Знаходиться у класі `DateRange` (перевірка валідності дат, логіка перетину інтервалів) та сутності `Room`.
- **Use Case:** Знаходиться у класі `BookingService`. Відповідає за оркестрацію (взяти номер з БД -> перевірити бізнес-логіку -> зберегти бронювання).
- **Ports / Interfaces:** Знаходяться у пакеті `application/port/`. `BookRoomUseCase` — вхідний порт (Driving), `BookingRepository` — вихідний порт (Driven).
- **Adapters:** Знаходяться у пакеті `infrastructure/adapter/`. `ConsoleController` — адаптер для симуляції дій користувача. `InMemoryRoomRepository` — адаптер сховища даних.

## 7. Інструкція запуску
1. Переконайтеся, що встановлено **Java 25** (або сумісну версію з підтримкою Records).
2. Склонуйте репозиторій.
3. Скомпілюйте проєкт або відкрийте його у середовищі розробки (наприклад, IntelliJ IDEA).
4. Запустіть файл `Main.java`. У консолі ви побачите результати виконання 4-х тестових сценаріїв (успішне бронювання, відмова через перетин дат, відмова через недоступність номера, відмова через хибний формат дат).
