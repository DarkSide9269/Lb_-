## Висновок

**Найкритичнішими проблемами у проєкті є вразливості з категорії Security (SQL Injection та логування паролів).** Зокрема, конкатенація SQL-запитів дозволяє зловмисникам маніпулювати базою даних (видаляти таблиці або обходити авторизацію). Такі помилки ставлять під загрозу всю систему та її конфіденційні дані, тому їх потрібно виправляти першочергово. 

На другому місці за критичністю — архітектурні порушення (God Class / логіка у контролері). Хоча код може працювати зараз, змішування UI-шару з доступом до БД робить подальше масштабування та тестування проєкту практично неможливим. Оновлення платформи до актуальної версії Java 25 та рознесення логіки по сервісах дозволить тримати кодову базу чистою та безпечною.
